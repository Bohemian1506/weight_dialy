#### 発火条件検証の論理構造 (= ブラックスワン論理 vs 確証の非対称性)

学び 36 サブパターン候補「3 箇所以上の同文反復 = 発火確率上昇」 を **単発実例で確証することは原理的に不可能** (= 確証性向の罠)。**反証は 1 件でできる** (= 5 箇所反復しても発火しなければ仮説却下方向)。本 PR は:

- **Day 12+ 後続 PR / dev-log PR / 3 者並列レビュー** で「3 mode 化提案」 「2 mode 確定の言い切りを格下げすべき」 等の指摘が出れば **発火確認 = 仮説支持データ +1**
- **指摘が出なければ仮説却下方向** (= 5 箇所反復しても発火しなかった = 「3 箇所以上反復」 でも発火確率上昇しない可能性)

**いずれにしても観測価値あり** = 仮説検証フェーズの正しい運用。

### 副次知見 2: design I-1 = 「別 Issue 切り出し + 本 PR 吸収」 の判断軽さ (= 観測モードに乗せない判断軸)

3 者並列レビュー指摘 9 件のうち、**design I-1 (= retry 連続失敗時の UX ガード) のみ別 Issue (#276) 切り出し**、残り 8 件は本 PR 吸収。判断軸は **2 軸** で機械的に行われた (= 軽量 Issue 1 PR ルール 12 例目達成の安定運用):

| 軸 | 判定 | 該当 |
|---|---|---|

| **本 PR スコープを超えるか** | Yes | design I-1 (= retryCount 状態管理 + 文言切替 + リセット 3 タイミング) → 別 Issue #276 |

**特に重要なのは I-1 を「観測モードに乗せなかった」 判断**。観測モードは **本 PR で確定する設計判断 (= 2 mode 化)** に対する反復で発火装置を仕掛けるが、I-1 は **本 PR 起票時に 1 文も触れていない外部観点 = 自己提唱なし = 学び 36 発火条件外**。観測モード乱用 (= レビュー由来の派生 Issue を全部本 PR に詰め込む) は **発火条件を曖昧化** するため、**「Issue 起票時に書いた言い切りに限り反復」** という運用境界を本セッションで暗黙確立。

### 副次知見 3: 軽量 Issue 1 PR ルール 12 例目連続達成 + Issue 削減フォーカスとの整合

#### 局長指示「Issue を減らすことに注力」 との整合

Issue 削減には **2 つの戦略** がある:

| 戦略 | 効果 | 副作用 |
|---|---|---|
| **A: 軽量から順消化** | 1 PR = 1 close で件数効率高、Closes 学習サイクル維持 | 大物 (= Tier 1) は残り続ける |
| **B: Tier 1 大物攻略** | 大物 1 件で複数子 Issue が連動 close | 件数は一時増える (= 1 partial 1 PR で増殖)、教材性は高い |

本セッションは **A 戦略採用** = 副局長判断で「Issue 件数を効率よく減らす」 文字通り解釈。Tier 1 大物 (= Phase 3 三重防衛 / Capacitor OAuth / daisyUI 全置換) は **件数増やしうる** 性質を局長に提示し回避決定。**この A vs B の判断軸自体が後輩教材として価値ある**。
