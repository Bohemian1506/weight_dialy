## 🧠 学び (= 副次知見、Day 13+ 観測候補)

### 副次知見 1: 不発閾値の非対称性 = 観測装置側要因の切り分け余裕

> ぜんぶ忘れていい、この 1 シーンだけ。
> 「3 例で確定 vs 5 例で却下」 の差は **観測装置の故障切り分け時間** を見ているだけ。実例 1 件あれば「動く装置」 と分かるが、何も起こらない時は「装置が壊れているのか仮説が間違いか」 区別がつかない、だから 2 例余分に観測する。

確定側 (= 3 例観測) の根拠 = Day 11-3 で 3 例目達成 + Day 11-4 で 4 例目実演 により「機械的に再現可能」 確定見え。

不発側 (= 5 例観測) の根拠 = 観測装置側要因 vs 仮説側要因の切り分けに余裕が要る。3 例不発の時点では装置設計ミス (= タイミング不一致 / セッション中断 / 仕様変更) を疑う段階、追加 2 例不発で「装置は健全、仮説が間違い」 が言える段階。

→ 「非対称性 = サンプル数の倍率」 ではなく **「不発の意味解釈に時間を払う」** という運用境界。

### 副次知見 2: 集計慣習の事前確定範囲 = 不発側まで含めて事前固定

Day 12-2 PR #293 で集計慣習の事前確定を確立した時点では「確定側 (= 3 例観測達成) を事前確定」 までだった。不発側は「不発が継続したら却下方向に記録」 と曖昧な表現で残っていた (= 不発閾値 N が未定義)。

→ 本 PR で **不発側まで事前確定範囲を拡張** = 「結果が出てから却下逃れバイアス」 を不発側でも回避。事前確定は確定側だけでなく不発側にも適用する原則を memory に明記。

### 副次知見 3: 2 段階自己適用 = Issue → memory 初稿 → 最終稿の 2 ステップ補正

通常の「Issue は仮説 / PR は確定、上書き可」 は 1 段補正 (= Issue → PR で 1 回整理)。本 PR では **2 段補正** が発生:

- 1 段目: Issue 起票 → memory 初稿 (= ブラックスワン論理逆応用 → 類似に補正)
- 2 段目: memory 初稿 → 最終稿 (= 3 者並列レビュー strategic 論点 1 で「類似もまだ正確でない」 指摘 → ブラックスワン枠組み削除、観測装置側要因切り分けだけで構成)

→ memory ルールの自己適用は **1 段で完結するとは限らない**、レビューで再相対化が起きれば 2 段、3 段と上書き連鎖。本 PR は 2 段で確定 (= 3 段目候補なし)。

### 副次知見 4: 数値 3/5 の自己提唱性 = 仮置き明示で予防接種 (= strategic 論点 2 反映)

### 副次知見 5: ハイゼンベルク的観測者問題 = 事前宣言ありの観測バイアス (= strategic 論点 3 反映)

## How to apply

- **不発閾値の事前確定原則**: 確定側だけでなく不発側もルール化前に固定する (= 結果が出てから却下逃れバイアスを回避)
- **観測装置側要因の切り分け**: 不発の意味解釈に時間を払う = 装置設計ミス vs 仮説側要因の区別のため、確定側より多めの観測 (= +2 例)
- **自己提唱の数値は仮置き明示で予防接種**: 独立根拠が弱い数値 (= 例: 不発閾値 3/5) は「仮置き、後で再評価候補」 を併記して学び 36 発火経路を 1 つ閉じる
- **memory ルールは 2 段以上の自己適用が起こり得る**: Issue → memory 初稿 → 最終稿、レビューで再相対化が起きれば 3 段、4 段と連鎖
- **事前宣言ありの観測は別軸記録**: 「観測装置として機能してほしい」 と依頼した瞬間に純粋偶発でなくなる (= ハイゼンベルク的観測者問題)、K0 純粋偶発 vs K1 事前宣言あり で 2 軸分割
