## 🔥 遭遇したトラブルと解決 (= メンテ)

### ターミナル高負荷 → Docker 不要キャッシュ掃除

ターミナルが重いとの観測から Docker ディスク使用状況を調査。`docker system df` で約 7GB の回収可能領域 + 旧プロジェクト成果物の蓄積を確認し、段階的に掃除:

| 対象 | Before | After | 備考 |
|---|---|---|---|
| Build Cache | 7.68GB | 0.59GB | 純キャッシュ、再生成のみ・データ損失なし |
| Images | 32 個 / 13.54GB | 2 個 / 2.74GB | 稼働中 weight_dialy 本体のみ残置 |
| Containers | 12 個 | 2 個 (稼働中のみ) | 5-8 ヶ月前 Exited の旧プロジェクト分を削除 |

- **実回収ディスク 約 9.2GB** (= Build Cache 7.1GB + 旧コンテナ/イメージ 2.1GB)。
- イメージ削除でビルドキャッシュ参照が外れ、追加で 5.17GB がキャッシュとして回収可能化 → 2 段階で掃除。
- **dangling ボリューム 14 個 (= 3.43GB) は保持**: 旧プロジェクトの DB データを含み、削除すると復元不可のため局長判断で残置。
- 教材ポイント: `docker system df` の RECLAIMABLE 列は共有レイヤを各イメージに重複計上するため、実際の回収量 (= prune コマンド出力の Total) より大きく見える。dev-log には実回収値 (9.2GB) を記載。
